# CI Hardening (#424) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every macOS job in `.github/workflows/test.yml` run on a pinned runner image + pinned Xcode, and give every job a uniform `timeout-minutes`, so the toolchain is deterministic and a hang can't burn 6h of runner time (#424).

**Architecture:** CI/config only — one file (`test.yml`) changes. Three edits: (1) pin the two Swift jobs (`swift-conformance`, `ios-host`) to `macos-26` + `setup-xcode@v1.7.0` selecting Xcode 26.5; (2) add `timeout-minutes: 30` to all six jobs; (3) pin `rust-test`'s macOS matrix leg to `macos-26`. No job's *behaviour* (commands, working-dir, matrix semantics) changes. The macOS-image/Xcode-pin behaviour can only be fully proven on live CI (the PR run).

**Tech Stack:** GitHub Actions YAML; `maxim-lobanov/setup-xcode`; `actionlint` (offline workflow linter) + `python3`/`pyyaml` for assertions; `swift test` (Swift 6.3) for the local no-regression check.

## Global Constraints

- **Pinned action SHAs only** — no moving tags. New pin: `maxim-lobanov/setup-xcode@ed7a3b1fda3918c0306d1b724322adc0b8cc0a90` **# v1.7.0**. Reuse existing pins verbatim (`actions/checkout@34e114876b0b11c390a56381ad16ebd13914f8d5 # v4`, `Swatinem/rust-cache@e18b497796c12c097a38f9edb9d0641fb99eee32 # v2`).

- **Uniform timeout:** `timeout-minutes: 30` on **every** job (a single value — no per-job drift). Provisional safety cap; re-tune against live CI later.
- **No `macos-latest` may remain in any `runs-on:` or matrix `os:` value** after the change.
- **CI/config only:** no `core`/crypto/FFI/on-disk-format change; no new error variant; `#![forbid(unsafe_code)]` intact; no production code. `test.yml` is the only file touched.

Edit/Write/Read MUST use the full `.worktrees/ci-hardening-424/...` path (a bare main-repo path silently hits `main` — see the edit-tool-targets-main memory).
