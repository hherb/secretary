## (3) Open decisions and risks

### Plan deviations (worth flagging to the reviewer)

1. **Placeholder icon at `desktop/src-tauri/icons/icon.png` was added.** The plan's Step 5 NOTE claimed the missing icon would only bite at `tauri build` time and that `cargo build` / `pnpm tauri dev` would tolerate its absence. In practice, `tauri::generate_context!()` is a proc-macro that **embeds the icon as a static asset at compile time** — `cargo build` fails with "failed to open icon … No such file or directory" if `bundle.icon[0]` resolves to a non-existent file, and `bundle.icon = []` does not help (Tauri 2 still attempts the default `icons/icon.png`). Resolved by generating a 32×32 solid dark-indigo placeholder with Python stdlib (`struct` + `zlib` only — no external deps) and committing it alongside [`desktop/src-tauri/icons/README.md`](../../desktop/src-tauri/icons/README.md) which documents the regeneration script and points at `pnpm tauri icon <source>` for real branding (a deferred D.1.x slice). The plan's NOTE in Task 1 Step 5 should be amended for future readers — minor doc drift, not worth a separate commit.
2. **Added `desktop/pnpm-workspace.yaml`** (not in the plan). pnpm 11 (current Homebrew `pnpm@11.3.0`) blocks all postinstall scripts by default; `esbuild`'s postinstall installs its platform-native binary, and without it Vite errors at first dev-server transformation. The legacy `"pnpm": { "onlyBuiltDependencies": [...] }` field in `package.json` is no longer read by pnpm 11. The new home is `pnpm-workspace.yaml` with `allowBuilds: { esbuild: true }`. Whitelist is intentionally minimal — any new entry requires explicit review.

Neither deviation changes the spec or the architectural decisions. Both are tooling-mechanics that the plan author couldn't have known without the live attempt.

### Decisions settled

- **Bundle identifier:** `org.secretary.desktop` (per spec). Can be revised pre-distribution (D.2 packaging).
- **CSP locked from day one** — `script-src 'self'`, `style-src 'self' 'unsafe-inline'` (the latter is required by some Svelte 5 emit modes; spec §11.4 explicitly accepts it).

### Risks carried forward

- **Real icon branding deferred to a later D.1.x slice.** Placeholder is fine for dev / packaging-disabled smoke tests, but `tauri build` for a release bundle will emit cosmetic warnings about missing platform-specific icon variants (no `.icns` for macOS, no `.ico` for Windows, no multi-size PNG set). When a brand asset exists, run `pnpm tauri icon path/to/source-1024x1024.png` from `desktop/src-tauri/` and update `tauri.conf.json` `bundle.icon` accordingly. Instructions live in `desktop/src-tauri/icons/README.md`.
- **First `cargo build` is slow (~45 s with all release-mode optimisations).** Subsequent incremental builds are seconds. Tauri 2's transitive graph (`tao`, `wry`, `tauri-codegen`, `tauri-macros`, …) is not small. Acknowledged tax.
- **Plan's gauntlet-count predictions for Tasks 2–5 should be re-validated.** Task 1 met its prediction exactly (960 → 960). The plan says Task 2 lands at 987 (+27), Task 3 at 999 (+12), Task 4 at 1002 (+3), Task 5 at 1005 (+3). If Task 2 lands at, say, 991, the surplus tests are good news but the plan's later predictions need a one-line note.
