### Architecture (where the pieces live)

- `browser/secretary-browser-host/src/frame.rs` — the load-bearing codec:
  4-byte native-endian length prefix + UTF-8 JSON, capped at `MAX_FRAME_LEN`
  (1 MiB), rejecting an oversized prefix **before** allocating the body. Every
  malformed-input path is a typed `FrameError`, never a panic; clean EOF at a
  frame boundary returns `Ok(None)` (shutdown).
- `browser/secretary-browser-host/src/protocol.rs` — `Inbound` (internally
  tagged, `Query` + a `#[serde(other)]` `Unknown` catch-all) and `Outbound`
  (`Available`/`Error`). `handle()` is pure and total: `Query → available{0}`,
  `Unknown → error`; every reply mints a fresh v4 `request_id`.
- `browser/secretary-browser-host/src/lib.rs::run()` — the loop, generic over
  `Read`/`Write` so `tests/echo.rs` drives it over an in-memory pipe. Bad JSON
  body (frame consumed) → error reply + keep serving; oversized prefix (body
  not consumed → stream misaligned) → error reply + stop; truncated/pipe `Io` →
  surfaced to `main`, non-zero exit.
- `browser/extension/` — MV3. `content.ts` (classic IIFE, no top-level
  import/export) collects `{top_origin, frame_origin, https}` and asks the
  worker; `background.ts` opens a one-shot `connectNative` port, forwards the
  `query`, logs `available`, disconnects. Scoped to `https://example.com/*`;
  `nativeMessaging` permission only.
- `browser/host-manifest/` — the dev manifest (`type: "stdio"`, `path` +
  `allowed_origins` placeholders) and the per-OS install + manual-smoke README.

- `desktop/src-tauri` won't build here — Tauri's `gdk-sys` needs GTK system libs

## (3) Guardrails that must keep holding

- **No `core/`, `ffi/`, `ios/`, `android/`, or on-disk-format change.** D.4.1
  was purely additive under `browser/` + `docs/` (+ the workspace-members line).
  D.4.2's crypto must go through the existing `open_with_device_secret` path —
  do not add a parallel open.
- **No listening socket** — the host speaks only over browser-provided stdio
  (threat-model §6 invariant 3). The manifest's `allowed_origins` binds the host
  to the extension ID.
- **Framing codec never panics and caps at 1 MiB.** Preserve the
  "assert `Result`, not panic" property if you extend `frame.rs`.
- `#![forbid(unsafe_code)]` + clippy `-D warnings` hold in the host crate.
