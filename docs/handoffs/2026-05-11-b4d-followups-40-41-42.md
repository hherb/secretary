## (3) Open decisions and risks

- **Swift + Kotlin smoke runners not executed this session** because the host is Linux without `swiftc` / `kotlinc`. The .udl change (`[Throws=VaultError]` on `owner_card_bytes()`) is verified at the binding-generator level via `uniffi-bindgen generate` for both targets — Swift renders `func ownerCardBytes() throws -> Data?` and Kotlin renders `@Throws(VaultException::class) fun ownerCardBytes(): kotlin.ByteArray?`. Source was updated for Swift's required `try` keyword at three call sites; Kotlin's unchecked-exception codegen requires no call-site change. PR #46 marks both as "**not run**" in its test plan; merging the PR should be gated on a macOS / Kotlin-equipped run of the smoke runners.
- **`signer_secret_keys()` accessor on `UnlockedIdentity` is now a single-caller-removed dead path.** With #42's refactor, the only previous caller (`share/orchestration.rs`) no longer invokes it. The accessor still exists (with `#[allow(dead_code)]`), and `SignerSecretKeysError` is still alive as part of its return type. Kept in place for forward-compat; revisit for deletion if no Sub-project-C surface picks it up. (Out of scope for #42; the comment on the accessor still references the old caller and is mildly stale — clean up next session if it bothers anyone.)
- **#41's `Err` arm is practically unreachable on the v1 invariant** — the card was decoded + verified at `open_vault` and lives behind an immutable handle, so re-encoding a previously-validated card is deterministic over fixed inputs. The `Result` widening is precautionary (issue #41's "future encoder version may grow non-trivial failure modes"), not a triggered fix. No new test exercises the `Err` path because no production call site can construct an unencodable card; the panic-replacement is verified by code review + Display-pin tests.
- **Issue #38 still open** — the `share_block` and `save_block` proptests are at 16 cases each because per-case Argon2id-protected vault open dominates wall-clock time. The cleanup pass this session does NOT address this; raising case counts requires a shared writable-vault fixture.
- **Bisect-friendliness:** all 4 commits each pass `cargo test --release --workspace` independently. The `share/orchestration.rs` file is touched in three commits (mapper for #40, doc-comment for #41, imports + Step 2 refactor for #42); the splits are clean and verified.
