The Sub-project C plan as written (ROADMAP §"Sub-project C") sequences C.1 → C.2 → C.3 → C.4. C.1.1a + C.1.1b are mid-C.1 refinements; C.1 itself is feature-complete for the pure-Rust state machine. There is no additional C.1.x slice queued. **Recommendation: start C.2 next.** The headless CLI is the natural validation harness for the merge layer and clears the path for D-phase platform UIs.

### Cross-language Python clean-room replay of `sync_kat.json`

[Issue #76](https://github.com/hherb/secretary/issues/76) — the Python clean-room verifier currently replays `conflict_kat.json` (CRDT merge semantics) but not `sync_kat.json` (sync state machine + merge layer). With C.1.1b ✅ and the KAT at 16 vectors across 4 vector-type families, this is now actionable. Two design options:

- **Option A** — bump `sync_kat.json` schema to v3 with fully self-describing fixture parameters so Python can rebuild fixtures byte-identically without re-encoding scenarios in code.
- **Option B** — re-encode the scenarios (`no_veto`, `single_veto`, `two_veto`, `stale_hash`, `fingerprint_mismatch`) as Python clean-room equivalents of the Rust fixture builders.

Decision deferred to the issue. Either way, the value is preserving the §15 "clean-room implementation from `docs/` alone" property as Sub-project C public surface grows.

### Risks (carried from C.1.1b design doc, restated for C.2)

- **CRDT proptests must not weaken.** Four properties in `core/tests/proptest.rs` (commutativity, associativity, idempotence, well-formedness) hold over the full record domain. C.2 must not touch `core/src/vault/conflict.rs` beyond bug fixes; if a CLI requirement seems to need a relaxation, push back and re-read the design doc.
- **`tempfile` exact pin** (`=3.27.0` in [`core/Cargo.toml`](../../core/Cargo.toml)) — do NOT bump as part of C.2 work. The pin guards §9 atomic-rename semantics under the manifest and block writes.
- **AEAD nonce per rewrite** — C.1.1b's per-test nonce constants (`CANONICAL_NONCE_A`, `SIBLING_NONCE_B/C/D`, `BLOCK_NONCE_E/F`) ensure key+nonce pairs are unique across fixtures. If C.2 adds new fixture flavours, follow the same pattern.
- **`#![forbid(unsafe_code)]`** — workspace-wide invariant. C.2 CLI must be pure-safe Rust; if a primitive truly needs FFI (e.g. platform-specific file watching beyond what `notify` covers), isolate behind a reviewed boundary.
